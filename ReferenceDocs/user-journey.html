Livestream Booking App — User Journey

Complete journey map for Brands, Creators, and Internal Team across the StreamMatch platform

1. System Overview

🏪

Brand (Seller)

Shops that want creators to host their livestreams. Submit applications specifying their requirements and slot needs. Identified by Shop ID + Shop Name.

🎙️

Creator (Affiliate)

Content creators who perform in brand livestreams. Browse available brands and apply with their preferred timeslots. Identified by Username + Phone.

🛠️

Internal Team (Admin)

Shopee staff who approve/reject both brand and creator applications, monitor confirmed slots, and manage platform configuration. Authenticated by password.


High-level flow: Brand applies → Internal team approves brand → Creators apply to brand with timeslots → Internal team approves creator → Creator confirms via Telegram or App → Slot confirmed ✅

3. Creator Journey

🎙️ Creator / Affiliate
1
Login / Registration
Landing page → choose "Creator" role → enter Affiliate Username + Phone. System creates or retrieves creator record.
  • First-time: redirected to Profile Setup (shipping address required)
  • Returning: land on Creator Dashboard
2
Browse Available Brands
Dashboard → "Available" tab. Shows all Approved brand applications in a list/calendar view.
  • Displays: brand name, stream month, slot count, category, AMS commission %, stream location etc.
  • Filter by category
  • Click any brand to view full details including the brief document
3
Apply with Timeslots
Click a brand → "Apply" → Submit timeslot proposals:
  • Creator Site brands: free-form picker — minimum 2 timeslots, always an even count (2, 4, 6…), each ≥ 2 hours, start date tomorrow or later within the brand's designated month
  • Seller Site brands: chip selector — creator picks from the brand's predefined timeslots (fixed 2-hour slots). Chips show as taken (grey, disabled) if already confirmed by another creator, or conflicting (yellow, disabled) if overlapping with the creator's own confirmed slots elsewhere. If all chips are taken, a "All Timeslots Filled" message is shown and no application can be submitted
  • System prevents overlap with other creators' slots (for same brand) and with your own confirmed slots at other brands — checked at chip selection and re-validated at submit on the backend (GAS) to guard against race conditions
Also fills in:
  • Telegram handle (required — used for approval notifications)
  • Shipping address (required — for sample delivery)
  • Willing to travel? (yes/no)
  • Using existing samples? (if yes, brand won't ship new ones)
Status after submit: Pending
4
Awaiting Internal Approval
Dashboard → "My Applications" tab shows status. Creator can cancel at any time before the slot is confirmed.

Receive Telegram Notification

Stream details shown → Two inline buttons:
[✅ Confirm Slot]  |  [❌ Reject Slot]
Can also confirm/reject from the dashboard "Pending My Confirmation" tab.

Confirm or Reject the Slot
Creator taps a button in Telegram (or acts on dashboard):

Confirmed — Track Slot & Samples
Dashboard → "Confirmed" tab shows all locked slots (list + calendar).

  • Tap "✅ Confirm Receipt" → sample status updates to Received

Telegram → Chat ID mapping: Creator's Telegram username is stored on first interaction and mapped to their Chat ID in the TelegramChatIds sheet. The bot uses webhook callbacks to process button taps and update Google Sheets in real time.

Deduplication: Telegram updates are deduplicated via update_id stored in PropertiesService (lastUpdateId) to prevent double-processing.

8. BusinessMappingValues Configuration

The BusinessMappingValues Google Sheet tab is the central config table for dropdown options and system settings. Changes take effect immediately on the next data load — no redeployment required.

Sheet Structure

ColumnDescription
TypeConfig category (e.g. VoucherTier, AvailableMonth)
CodeInternal value stored in the database (e.g. tier1, 2026-05)
DescriptionHuman-readable label shown in the UI (e.g. Tier 1 (10%), May 2026)
ActiveSet to ACTIVE to enable; INACTIVE to disable

Configurable Fields

TypeUsed ForExample RowIf Not Configured
AvailableMonth Controls which months appear in the brand application form's "Livestream Period" dropdown Type=AvailableMonth · Code=2026-05 · Description=May 2026 · Active=ACTIVE Brand form month dropdown shows no options — brands cannot submit applications
VoucherTier Controls the voucher tier dropdown in the brand application form Type=VoucherTier · Code=tier1 · Description=Tier 1 (10%) · Active=ACTIVE Voucher tier dropdown shows no options — brands cannot select a tier
ProductNominationLink Controls the product nomination link to be shown in the brand application based on the stream month selected Type=ProductNominationLink · Code=2026-04 · Description=https://seller.shopee.sg/portal/marketing/cmt-live-stream/campaign/2048114986/session/10000000044414 · Active=ACTIVE Brand form production nomination link shows as https://seller.shopee.sg/portal/marketing/cmt-live-stream/campaign/Inset
InternalPIC Controls the PIC name shown on the dashboard so that brands / creators can reach out if they face issues Type=InternalPIC · Code=+6590000000 · Description=Kayla Ang · Active=ACTIVE Will default to Kayla's contact details
BrandActivation Controls the Brand Activation options available for brands to select in their application Type=BrandActivation · Code=SBD · Description=Super Brand Day · Active=ACTIVE Will be empty

How to Update

  1. Open the Google Sheet → BusinessMappingValues tab
  2. To add a new entry: append a new row with all four columns filled, Active=ACTIVE
  3. To disable an entry: clear the Active column value (leave blank) or put as INACTIVE
  4. To re-enable: set Active back to ACTIVE
  5. Changes are live on the next page load — no code changes or redeployment needed
Dashboard display is independent of AvailableMonth config. All dashboards (brand, creator, internal) always show current month onwards regardless of what is configured in BusinessMappingValues. The AvailableMonth config only controls which months brands can apply for in the form.

9. Architecture Quick Reference - For BPM Reference

Frontend

  • shopee-live-creator-match.html — full brand + creator + internal system
  • livestream-booking.html — simple slot booking UI
  • Session state in sessionStorage (key: lbSession)

Backend

  • livestream-booking-backend.gs — Google Apps Script
  • REST API via doGet(e) (GET-only, avoids CORS preflight)
  • doPost(e) for Telegram webhook
  • keepWarm trigger runs every 5 min on HEAD deployment

Data

  • Google Sheets: Sellers, Affiliates, Slots, Bookings, BrandApplications, CreatorApplications, TelegramUsers, InternalTeam, RMs, BusinessMappingValues
  • All data stored as plain text (@ format) to prevent date auto-formatting
  • 3-month rolling data window for most views
